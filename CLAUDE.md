# CLAUDE.md — Harragan Driveway Designs

## Project Overview

Business marketing website for **Andy Harragan & Sons LTD**, a driveway design and construction company. The site showcases services (resin work, block paving, driveways, tarmac/shingle, stone carpets), features a blog, and includes a contact form. Built with the Lovable AI platform.

---

## Tech Stack

| Layer | Tool |
|---|---|
| Framework | React 18 + TypeScript |
| Build | Vite 5 (`@vitejs/plugin-react-swc`) |
| Styling | Tailwind CSS 3 + custom HSL CSS variables |
| UI Components | shadcn/ui (Radix UI primitives) |
| Routing | React Router DOM v6 |
| Forms | React Hook Form + Zod |
| Data Fetching | TanStack React Query 5 |
| Animation | Framer Motion |
| Unit Tests | Vitest + Testing Library |
| E2E Tests | Playwright |
| Linting | ESLint 9 (TypeScript + React Hooks plugins) |
| Formatting | Prettier |
| Package Manager | npm (Bun lockfile also present but npm is primary) |

## Code Style & Conventions

- **TypeScript:** Relaxed config — `strict: false`, `noImplicitAny: false`, `strictNullChecks: false`. Don't add strict annotations unless explicitly asked.
- **Path alias:** `@/` resolves to `src/`. Always use `@/` imports, never relative `../../`.
- **Class merging:** Use the `cn()` helper from `@/lib/utils` for conditional Tailwind classes.
- **Styling:** Tailwind utility classes only — no CSS modules, no inline styles. Theme colors are CSS variables (`bg-primary`, `text-foreground`, etc.) defined in `tailwind.config.ts`.
- **Fonts:** `font-display` (Outfit) for headings, `font-body` (Inter) for body text.
- **Components:** Functional components with TypeScript. Props interfaces declared inline or immediately above the component.
- **shadcn/ui:** Components in `src/components/ui/` are generated by the shadcn CLI — prefer not to hand-edit them. Add new shadcn components via `npx shadcn-ui@latest add <component>`.
- **Forms:** React Hook Form + Zod schema validation. See `Contact.tsx` for the pattern.
- **No linting rule for unused vars** — `@typescript-eslint/no-unused-vars` is disabled.

## Constraints & Gotchas

- **Dev server runs on port 8080**, not the default 3000/5173. HMR overlay is disabled in vite config.
- **No backend / API.** This is a purely static front-end site. The contact form does not currently wire to a real endpoint — check `Contact.tsx` before assuming form submission works.
- **Blog is static data**, not a CMS. All posts live in `src/data/blogPosts.ts`. Adding a post means editing that file.
- **No environment variables** are used or required at this time.
- **`ServicePageTemplate.tsx` is very large** (37 KB). Read it carefully before modifying — it drives all service detail pages.
- **`components/ui/`** files are shadcn-generated. Treat as third-party; customize via Tailwind variants or wrapper components rather than editing the source.
- **Both npm and Bun lockfiles exist** (`package-lock.json` and `bun.lock`). Use `npm` for consistency unless instructed otherwise.
- **TypeScript is intentionally loose.** Don't tighten type-checking settings without explicit instruction — the project was built this way intentionally.
- **`download_images.py` and `parse.js`** in the root are utility scripts unrelated to the web app build — safe to ignore.
- **`lovable-tagger`** dev dependency is a Lovable platform hook for the AI editor — it's harmless but should not be removed.
